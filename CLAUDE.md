## Repository Purpose

This repository is the **living wiki** for Chroxy — a self-hosted remote terminal and chat client for AI coding CLIs (Claude Code, Gemini, Codex), reachable from a phone or desktop over a secure Cloudflare tunnel. It is a Quartz 4 site over an Obsidian vault in `content/`, deployed to GitHub Pages and mounted at `https://www.blamechris.com/chroxy-docs/`.

## Source of Truth

The canonical documentation lives in the **main chroxy repo** at [github.com/blamechris/chroxy](https://github.com/blamechris/chroxy) — its `README.md`, `CLAUDE.md`, `packages/server/CONFIG.md`, and the `docs/` tree (architecture reference, providers, security threat models, skills, container isolation). This wiki mirrors them; it should never hold a documented behaviour that isn't also true in the main repo.

**When behaviour changes:** update the main repo's docs/code first, then mirror it here.

## Documentation Sync Protocol

Sync this wiki after merging PRs in the main repo that change documented behaviour:

| Main repo change | Docs to update |
| ---------------- | -------------- |
| New / changed provider or capability | `content/providers.md` |
| Server, tunnel, WS protocol, or data-flow change | `content/architecture.md` |
| New config key, cost/timeout, or precedence change | `content/configuration.md` |
| Auth, token, encryption, or permission-gating change | `content/security.md` |
| Docker / sandbox / worktree / environments change | `content/container-isolation.md` |
| Skills loading, allowlist, or trust change | `content/skills.md` |
| Repo layout, dev/test commands, or extension surface | `content/developer-guide.md` |
| Anything user-facing on the landing page | `content/index.md` |

**Sync workflow:**
1. Check the main chroxy repo's recent commits and the relevant `docs/` files.
2. Identify feature PRs since the last docs update.
3. Read the relevant source files for accurate details.
4. Update the affected wiki pages.
5. Commit and push to deploy.

## Tech Stack

- **Quartz 4.x** — static site generator
- **Obsidian** — Markdown editing (vault is `content/`)
- **GitHub Pages** — hosting, custom domain `www.blamechris.com`

## Development

```bash
npm ci
npx quartz build --serve   # local preview at http://localhost:8080
npx quartz build           # production build into public/
```

## Deployment

Pushing to `main` triggers the Quartz deploy workflow in `.github/workflows/`:
1. Build the Quartz site
2. Deploy to the `gh-pages` branch
3. Live at https://www.blamechris.com/chroxy-docs/
